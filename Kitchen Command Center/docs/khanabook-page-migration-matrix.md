## Notes for the Angular team

1. **No new endpoints.** Every screen in the redesign consumes payloads that
   already exist. If a UI element seems to need new data, prefer deriving it
   client-side from an existing endpoint over adding a backend field.
2. **Guards + routes unchanged.** `LimitedAccessGuard`, role guards,
   `AuthInterceptor`, and route paths remain as-is.
3. **Marketplace credentials.** Never render `••••••••` inside an
   `<input>`. Use `KbSecretInputComponent`; its stored state is a display
   element, and the Replace action swaps it for an editable input.
4. **Temp password / recovery token.** Both are single-view artefacts:
   never persisted in local storage, never shown twice, always paired with
   a copy control and a "share privately" warning.
5. **Refund flow.** Preserve the two-step Review → Confirm. Duplicate
   submission must be blocked at the component level (disable + spinner)
   before it reaches `refundOrder`.
