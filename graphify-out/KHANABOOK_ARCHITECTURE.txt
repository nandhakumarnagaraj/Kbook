================================================================================
KEY COMPONENTS BREAKDOWN
================================================================================

ANDROID POS APP (Jetpack Compose)
─────────────────────────────────
• Target: API 26-36 (mobile + tablet)
• Screens implemented:
  - HomeScreen.kt         → Dashboard with metrics (Orders/Revenue/Avg)
  - NewBillScreen.kt      → Bill creation with menu items, cart, qty controls
  - OrdersScreen.kt       → Order list with filter chips, status tracking
  - MenuConfigurationScreen.kt → Category/item management
  - EasebuzzPaymentScreen.kt   → Payment processing (Cash/UPI/Card/Easebuzz)
  - SplashScreen.kt, LoginScreen.kt, SignUpScreen.kt
  - OtpVerificationScreen.kt, InitialSyncScreen.kt
  - EasebuzzKycScreen.kt, MarketplaceOrdersScreen.kt
  - ReportsScreen.kt
• Uses Premium Saffron design system (#F97316 primary)
• Features: Offline mode, sync, printer integration, GST/FSSAI lookup

WEB ADMIN DASHBOARD (Angular 18)
────────────────────────────────
• Platform Dashboard    → Business metrics, sub-merchant management
• Business Dashboard    → Orders, menu, storefront management
• Sub-Merchants Page  → CRUD with KYC aging column, action buttons
• Marketplace Setup     → Zomato/Swiggy integration
• Tax Compliance        → GST reports, CSV export
• Login with brand logo, dark/light mode support

SERVER (Spring Boot)
────────────────────
• Base Path: /api/v2
• Key Services:
  - EasebuzzApiClient.java (9+ payment APIs)
  - SubMerchantService.java (KYC, split labels, payouts)
  - PaymentRoutingService.java (intelligent routing)
  - ChargebackPreventionService.java (velocity checks)
  - CustomerDataService.java (privacy compliance)
  - Webhook endpoints (UrbanPiper/Swiggy/Zomato)

• Key Entities:
  - EasebuzzSubMerchant (+branchName, splitLabel)
  - Bill, BillItem, BillPayment
  - CustomerProfile (with opt-out support)
  - MarketplaceOrder (with integration status)
