# Kbook Project Rules

## Git Workflow
- Always commit directly to `main`. Never create feature branches.
- Do not create or switch to any branch other than `main`.
- All changes go straight to `main` and are pushed to `origin/main`.

## Available Skills

- `daily-prod-health-report` — Run comprehensive production health audit (OS, Docker, DB, traffic, security, SSL, backups, scoring, per-request latency, connection health). Place in `.claude/skills/` directory.
