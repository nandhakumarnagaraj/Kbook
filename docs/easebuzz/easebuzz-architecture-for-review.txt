1. WHAT WE'RE BUILDING
================================================================================

KhanaBook is a restaurant POS + ordering platform. We use Easebuzz as our
payment gateway with a marketplace model:

  - One parent merchant account (ADNX3KYX5)
  - Each restaurant onboarded as a sub-merchant
  - Payments route to sub-merchant's bank account via split labels
  - Admin web UI manages the full sub-merchant lifecycle

4. SUB-MERCHANT LIFECYCLE (OUR FLOW)
================================================================================

  1. Admin creates sub-merchant DRAFT in KhanaBook DB
       ↓
  2. Admin clicks "Submit to Easebuzz"
       ↓ POST /merchant/v1/submerchant/create/
       ↓ Easebuzz returns submerchant_id (e.g. S280498SF77)
       ↓ Status set to PENDING_KYC
       ↓
  3. Admin clicks "Generate KYC Link"
       ↓ POST /submerchant/v1/generate_kyc_access_key
       ↓ Easebuzz returns KYC portal URL
       ↓ Restaurant owner uploads documents via KYC portal
       ↓
  4. Easebuzz webhook POSTs KYC status to our server
       ↓ POST /api/v2/payments/easebuzz/sub-merchant/webhook
       ↓ We update sub-merchant status to ACTIVE
       ↓
  5. Admin clicks "Create Split Label"
       ↓ POST /split/v1/create
       ↓ Label sm_{submerchant_id} created
       ↓
  6. Payments now route through this sub-merchant
       ↓ POST /payment/initiateLink (with sub_merchant_id)
       ↓ After payment, post-split creates distribution
